筆記 | TypeScript - Object.keys 為什麼不安全

發表於 2022-11-26 分類於 Front-end ， TypeScript
文章字數： 4.6k 所需閱讀時間 ≈ 4 分鐘

影片來源 : Object.keys is weird, but DELIBERATELY - YouTube
推特討論串 : https://twitter.com/mattpocockuk/status/1596167864553459715

摘要

Object.keys 會回傳物件的 key 陣列，照理來說要是一定範圍的值，但是 Object.keys 的回傳值卻是 string[] ，即便我們知道陣列內的值一定都來自 object 所有的 key。另外如果又因為透過 Object.keys 變化後產生新的 Object (例如 : Object.keys(object).reduce(()=>,{}) )，新 key 想根據轉變的 key 賦予新值，這會導致 VScode 並不會提示這邊的錯誤，即使加上型別，這也是為什麼說 Object.keys 是不安全的。

不安全好像說不能用一樣，應該說不那麼精準。

原始的錯誤提示

ts playground : TypeScript: TS Playground

再生新的 object 不會提示錯誤

透過額外的 function 在包裝 object 產生新的 object ，這樣會使得原本的 IPerson 型別被覆蓋過去，即使指定型別也可以跨過限制。

newObject 照理說是按照 IPerson 定義的型別，預期這裡會跳錯誤，但是 沒有。IPerson 的規則就會略過了，看起來很像失靈 XD。

覆蓋解決 : 利用函式泛型來回傳

下面留言有人使用泛型的方式 :

我在 TS playground 使用時 typedKeys()， .keys(obj) 的 obj 會出現紅線

No overload matches this call.
Overload 1 of 2, ‘(o: {}): string[]’, gave the following error.
Argument of type ‘O’ is not assignable to parameter of type ‘{}’.
Overload 2 of 2, ‘(o: object): string[]’, gave the following error.
Argument of type ‘O’ is not assignable to parameter of type ‘object’.(2769)
input.tsx(30, 20): This type parameter might need an extends {} constraint.
input.tsx(30, 20): This type parameter might need an extends object constraint.

我不太知道是為什麼🤔，當下使用的版本是 v4.8.4，後來我降版到 v4.7.4 紅線會取消。是不是因為是泛型要檢查是不是物件 ? 因此我使用 typeof 檢查，結果還是不通過。

阿對啦 ! 別忘了 JavaScript 的 null 也是 “object”，所以不能這樣，那改成 !obj ?

很好這樣不會出現錯誤，但是有另一種方法是 v.4.8.4 針對 null object type 解決擴充
參考 :

  1. Announcing TypeScript 4.8 - TypeScript
  2. Improve intersection reduction and CFA for truthy, equality, and typeof checks by ahejlsberg · Pull Request #49119 · microsoft/TypeScript · GitHub
  • 改成 : O & {} 就解決了

不過這兩種顯示內容不太一樣 :

  1. !obj : (keyof IPerson)[] | undefined
  2. O&{} : (keyof IPerson)[]

可以看出 O & {} 更嚴謹。

在 twitter 討論串的 回答 中也有類似的答案 :

由於內容寫 Obj obj 有點混淆，我改成 O。

實際範例

我很常用 Object.entries 、Object.keys 結合 Array.reduce 產生新的物件，在寫 TypyScript 時常常碰壁，範例來自我寫 styled-components 的響應式字串 :

  1. as Array<keyof typeof breakpointSizes> : keys 產生的陣列型別轉型成 breakpointsSizes 的 key 的型別，如此一來就會按照 "xxl" | "xl" | "lg"... 。
  2. as Record<keyof typeof breakpointSizes, string> : 在最後產生的 object 手動添加型別
